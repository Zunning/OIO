# OIO 开发日志

## 2026-07-02：第一版完成

### 起点

项目最初目标是做一个个人英文表达学习工具。

最初设想的学习流程：

```text
先输出中文 / 英文 / 中英混合内容
-> 用 AI 改写成自然英文
-> 粘贴到软件里
-> 高亮表达
-> 挖空
-> 生成卡片复习
```

第一版决定不接 AI API，而是先做手动粘贴版本。

### 关键产品判断

第一版开发中逐步确定了几个核心判断：

- 高亮对象应该是完整 `chunk`，不是直接等于挖空答案。
- 挖空应该发生在 chunk 内，可以只挖 chunk 的一部分。
- 长文本和零碎表达都要支持，所以复习要按同一文本语境分组。
- 一张卡片只测试一个明确表达点，避免判断标准模糊。
- 复习时要保留语境，但不能提前泄露答案。
- 原始输入中的“我想说但说不出”也很重要，应该先标出来。

### 已实现功能

第一版已经实现：

- 本地静态 Web App
- 文本新建、编辑、删除、搜索
- 文本日期
- 文本类型，默认 `dialogue`
- 原始输入高亮 Source Intent
- 原始意图备注
- 原始输入中按 `Enter` 快捷保存意图
- 原始输入中半个英文单词自动扩展到完整单词
- 英文文本高亮 chunk
- 英文文本中按 `Enter` 快捷创建 chunk
- 英文文本中半个英文单词自动扩展到完整单词
- 创建 chunk 后自动进入挖空界面
- chunk 内选择单词挖空
- 支持多个空
- 支持非连续空
- 生成卡片
- 卡片绑定 Source Intent
- 卡片库
- 按文本分组复习
- 复习时输入答案
- 拼写检查
- 复习评分 Again / Good / Easy
- 可滚动英文上下文
- 上下文隐藏答案
- 复习时显示完整 chunk 的挖空高亮
- 复习时可显示原始输入并高亮 Source Intent

### 重要修正

#### 1. Chunk 与 Cloze 分离

最初高亮表达容易被理解成“直接挖掉整段”。

后来明确：

```text
chunk = 值得学习的完整表达块
cloze = chunk 内真正要测试的词
```

这个改动让卡片更小、更可判断。

#### 2. 复习时不能泄露答案

早期复习页在下方显示了完整 chunk，导致答案会直接出现。

后续改成：

```text
显示完整 chunk 的挖空版本
上下文中也隐藏答案
```

#### 3. 多个挖空的 bug

早期挖空逻辑用完整答案字符串替换句子。

当用户选择多个不连续单词时，完整答案字符串并不存在于原句中，所以复习时不会显示空。

修复后：

```text
按 chunk 内 token index 生成 maskedChunk
卡片保存 maskedChunk
复习时优先使用 maskedChunk
```

#### 4. Source Intent

新增原始输入高亮功能后，OIO 的学习闭环更完整：

```text
原始意图
-> 英文 chunk
-> 挖空卡片
-> 语境复习
```

这让复习不只是“看到英文补英文”，而是更接近“从自己的表达意图调出自然英文”。

### 当前技术形态

当前是无构建静态版：

```text
index.html
styles.css
app.js
```

数据存储：

```text
localStorage
```

优点：

- 不需要安装依赖
- 双击即可运行
- 适合快速试用和迭代

限制：

- 数据备份能力弱
- 代码会随功能增长变大
- 后续复杂功能更适合迁移到 React / IndexedDB / SQLite

### 验证

开发过程中做过静态加载检查：

```text
app.js 可以在带 DOM stub 的环境中正常解析和初始执行
```

由于当前环境没有可调用的浏览器命令，也没有 Node/npm 项目依赖，所以没有进行完整浏览器自动化测试。

### 第一版结论

OIO 第一版已经完成，可以开始日常使用。

它已经打通：

```text
原始表达意图
-> 自然英文 chunk
-> chunk 内挖空
-> 输入答案复习
-> 语境回看
```

下一阶段重点应该先放在数据可靠性和复习体验，而不是马上接 AI API。

## 2026-07-03 第二版完成日志

第二版主要目标：

```text
让 OIO 从“能用的第一版”
升级成“可以比较舒服地日常使用的个人学习工具”
```

这一版没有接 AI API，仍然保持手动复制 ChatGPT 的工作流，但明显增强了：

- 复习调度
- 搜索
- 导出
- 图片语境
- 原始意图匹配
- 挖空效率

### 1. 复习系统 V2

新增复习系统基础版：

```text
重要 / 普通
每组复习数量
自动排序
连续 Easy 拉长间隔
三次成功后毕业
复习完成总结
继续复习
熟卡清理
签到日历
```

当前复习排序逻辑：

```text
重要优先
掌握次数少优先
复习次数少优先
新学内容优先
更早到期作为兜底排序
```

卡片毕业规则：

```text
Good / Easy 累计 3 次成功
-> 标记毕业
-> 不再进入普通复习

Again 不计入成功次数
```

Easy 间隔：

```text
连续 Easy 会逐步拉长到 3 / 7 / 14 / 30 天
```

### 2. 今日复习与签到

首页增加：

```text
今日复习
连续复习天数
签到日历
```

点击日历中的某一天时，不直接铺开所有卡片，而是先显示：

```text
当天复习了哪些文本组
```

再点击某个文本组，查看该组当天复习过的 chunks。

这样更适合之后做：

```text
按组导出
按组复盘
按组生成学习材料
```

### 3. 文本导出

文本库增加批量导出：

```text
选择文本
选择格式
选择字段
复制导出内容
下载导出文件
```

支持格式：

```text
Markdown
HTML
纯文本
```

支持字段：

```text
标题
日期
原文
改写
翻译
图片
```

下载文件规则：

```text
Markdown -> .md
HTML -> .html
纯文本 -> .txt
```

### 4. 图片语境

每个文本可以上传一张或多张图片。

使用场景：

- 截图
- 聊天记录
- 网页材料
- 书页
- 生活场景照片

图片现在可以出现在：

```text
文本详情页
新建 / 编辑文本窗口
复习上下文
HTML / Markdown 导出
```

实现方式：

```text
图片转 dataUrl
存入 localStorage
```

当前限制：

```text
图片多了会让 localStorage 变大
后续更适合迁移到 IndexedDB
```

### 5. 搜索优化

修复搜索框输入时失焦的问题。

新增搜索范围：

```text
全部
标签
文本
标题
```

搜索范围含义：

```text
全部：标题、原文、改写、翻译、备注、标签
标签：只搜索标签
文本：原文、改写、翻译、备注
标题：只搜索标题
```

### 6. Source Intent 匹配优化

第一版中，卡片绑定原始意图是在挖空界面里选择。

第二版改成：

```text
高亮英文 chunk
-> 立刻弹出原始意图匹配
-> 选择一个原始意图
-> 再进入挖空界面
```

并且：

```text
一个原始意图被绑定后
默认不再出现在下一个 chunk 的匹配列表里
```

如果原始意图已经全部绑定完：

```text
不再弹匹配窗口
直接进入挖空界面
```

这个流程更符合实际学习路径：

```text
我原来想说但不会说什么
-> AI 是怎么自然表达的
-> 哪个词需要挖空复习
```

### 7. 挖空效率优化

挖空界面保留原来的单击多选：

```text
单击 token
-> 选择 / 取消
-> 适合多词表达
```

新增双击快速保存：

```text
双击某个单词
-> 直接生成卡片
-> 关闭挖空窗口
```

这样单词级挖空会更快，多词表达仍然可以用原来的保存按钮。

### 9. 当前边界

第二版仍然是静态本地版：

```text
index.html
styles.css
app.js
localStorage
```

优点：

- 快速
- 本地
- 不依赖后端
- 不花 API 钱

限制：

- 图片和数据都存在 localStorage，长期大量使用会有容量压力
- 没有自动备份
- 没有数据导入/恢复工具
- 没有正式测试框架
- 代码已经开始变大，后续复杂功能可能需要拆分

### 10. 第二版结论

第二版已经可以作为日常学习工具使用。

现在 OIO 的核心闭环是：

```text
原始表达
-> AI 改写
-> 原始意图高亮
-> 英文 chunk
-> 原始意图匹配
-> chunk 内挖空
-> 卡片复习
-> 复习总结
-> 签到记录
-> 文本 / 图片导出
```

下一阶段更适合优先考虑：

```text
数据备份 / 导入
IndexedDB
按组导出
图片导出增强
音频 / 听音练习
```
