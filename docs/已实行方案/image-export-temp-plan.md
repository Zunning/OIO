# 图片上传与导出临时方案

## 1. 目标

给每一个文本增加图片附件能力：

```text
一个文本
-> 原始输入
-> 英文改写
-> 中文翻译
-> 1 张或多张图片
```

导出文本时，可以选择是否把图片也一起导出。

这个功能主要解决两个问题：

- 有些表达来自截图、聊天记录、网页、书页或生活场景，只保存文字会丢失语境
- 导出时可以把图片和文本一起带出去，方便做笔记、复盘、整理材料或以后做图片卡片

## 2. 第一版范围

第一版只做轻量版本：

```text
文本详情里上传图片
文本详情里查看图片
编辑文本时保留图片
删除文本时删除对应图片
批量导出文本时可以勾选图片
```

暂时不做：

- 图片裁剪
- 图片标注
- OCR 识别图片文字
- 图片生成卡片
- 图片参与复习调度
- 批量导出 PNG 长图

## 3. 数据结构

建议新增 `images` 数据：

```ts
type TextImage = {
  id: string;
  textId: string;
  name: string;
  dataUrl: string;
  createdAt: string;
};
```

字段说明：

- `textId`：属于哪一篇文本
- `name`：原始文件名，导出时可显示
- `dataUrl`：图片本体，第一版直接存在 localStorage
- `createdAt`：上传时间

第一版可以直接放进当前 `localStorage`，实现最简单。

但要注意：图片会明显变大。后续如果图片多，应该迁移到 IndexedDB。

## 4. 上传体验

在文本详情页增加图片区：

```text
图片
[上传图片]

[缩略图 1] [缩略图 2] [缩略图 3]
```

交互：

- 支持一次选择多张图片
- 上传后显示缩略图
- 点击缩略图可以放大查看
- 每张图片可以删除

限制建议：

```text
单张图片建议压缩到 1200px 宽以内
单张图片尽量小于 1MB
```

第一版可以先不做复杂压缩，只做文件大小提醒。

## 5. 导出体验

当前导出功能已经可以选择：

```text
标题
日期
原文
改写
翻译
```

可以新增：

```text
图片
```

导出格式先分两种：

### 5.1 Markdown 导出

如果勾选图片：

```md
## 标题

**Date:** 2026/07/03

**Rewrite**

...

**Images**

![image-1](data:image/png;base64,...)
![image-2](data:image/jpeg;base64,...)
```

优点：

- 最容易实现
- 可以复制出去
- 图片可以跟随文本一起保留

缺点：

- dataUrl 很长，导出的 Markdown 会变大
- 有些软件不一定支持 dataUrl 图片

### 5.2 纯文本导出

纯文本无法真正包含图片，只能导出图片信息：

```text
Images:
- screenshot-1.png
- photo-2.jpg
```

所以如果用户选择纯文本 + 图片，可以提示：

```text
纯文本只能导出图片文件名，不能嵌入图片。
```

## 6. 后续更好的导出方式

后续如果要做得更完整，可以加：

```text
导出 HTML
导出 ZIP
导出长图 PNG
```

推荐优先级：

```text
1. Markdown + dataUrl 图片
2. HTML 导出
3. ZIP：文本文件 + 图片文件
4. PNG 长图
```

其中 ZIP 会更适合真正保存图片，因为图片可以作为独立文件导出。

## 7. UI 位置

### 7.1 文本详情页

放在右侧栏或正文下方：

```text
图片
[上传图片]
[缩略图列表]
```

### 7.2 文本库导出区

在字段选择里加：

```text
[ ] 图片
```

如果当前选择的文本里没有图片，可以仍然显示选项，但导出时跳过空图片。

## 8. 实现顺序

### 阶段一：图片数据和上传

- `initialState` 增加 `images: []`
- 新增 `imagesForText(textId)`
- 文本详情页显示图片区
- 上传图片转成 dataUrl 存储
- 删除图片
- 删除文本时同步删除图片

### 阶段二：图片预览

- 缩略图列表
- 点击图片打开预览 modal
- modal 内显示大图和文件名

### 阶段三：导出图片

- `exportOptions` 增加 `images`
- 导出面板增加“图片”选项
- Markdown 导出嵌入 dataUrl 图片
- 纯文本导出图片文件名

### 阶段四：优化存储

- 图片过大时提示
- 后续考虑压缩
- 后续考虑 IndexedDB

## 9. 推荐第一版实现

第一版建议做到：

```text
上传多张图片
文本详情显示缩略图
点击放大
删除图片
导出时可勾选图片
Markdown 中嵌入图片 dataUrl
纯文本中列出图片名
```

不建议第一版就做 ZIP 或 PNG 长图，因为会把实现复杂度拉高。

## 10. 小结

图片功能应该先作为“文本语境附件”存在，而不是复习核心。

这样它不会干扰现在已经稳定的：

- 文本
- chunk
- 挖空
- 卡片
- 复习
- 签到
- 导出

等图片附件稳定之后，再考虑：

- 图片卡片
- OCR
- 按组生成复盘图
- 批量导出图片材料
