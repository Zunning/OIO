# OIO 使用指南

## 1. OIO 是什么

OIO 是一个个人英文表达学习工具。

它不是传统的背单词软件，也不是单纯的 AI 改写记录本。它的核心是一个闭环：

```text
Output：先从自己的真实生活、想法和情绪出发
Input：把 AI 改写后的自然英文变成自己的语境材料
Output：复习、内化之后，再回到真实生活里用出来
```

换句话说，OIO 关注的不是“别人会说什么”，而是：

```text
我本来想说什么？
自然英文里可以怎么说？
我怎么把这个表达变成下次真的能用出来的东西？
```

所以 OIO 的学习单位不是孤立单词，而是和你自己的语境绑定的 `chunk`。一个 chunk 可以是一个词块、短语、句型，也可以是一小段你觉得以后会反复用到的表达。

## 2. 打开方式

OIO 当前是本地静态版。

直接打开项目根目录里的：

```text
index.html
```

即可使用。

当前数据保存在浏览器的 `localStorage` 里，建议固定使用同一个浏览器打开。换浏览器、清理浏览器数据或更换电脑，都可能导致看不到原来的数据。

## 3. 推荐学习流程

日常推荐流程：

```text
写下原始输入
-> 用 OIO Prompt 发给 ChatGPT
-> 把 ChatGPT 的 OIO 格式结果粘贴回来并自动解析
-> 标出原始输入里“想说但说不出来”的地方
-> 在英文改写里高亮对应 chunk
-> 匹配原始意图
-> 在 chunk 里挖空生成卡片
-> 在语境中复习
-> 回到生活里再次输出
```

这个流程的重点是：先让大脑知道“我缺的是哪个表达”，再去看 AI 是怎么自然表达的，最后用复习把它变成自己的可调用材料。

## 4. 使用 Prompt 生成文本

进入 `Prompts` 页面，可以选择：

- Everyday Expression / IELTS Training
- 英式英语 / 美式英语
- CEFR 难度或 IELTS 分数
- 是否允许 emoji
- dialogue / writing / work / casual / story 等风格

把自己的原始输入写进去后，点击：

```text
复制提示词 + 原始输入
```

再粘贴到 ChatGPT。

ChatGPT 返回 `:::OIO ... :::` 格式后，回到 OIO 的新建文本窗口，把内容粘贴到 `AI 输出`，点击：

```text
自动解析
```

如果勾选了：

```text
解析后自动保存
```

OIO 会在解析成功后直接保存文本。这个勾选状态会被记住，下次打开不用重新勾。

保存文本后，英文改写会自动复制到剪贴板，方便你继续使用或发送给别人。

## 5. 新建文本

点击：

```text
+ 新建文本
```

可以手动填写，也可以通过 AI 输出自动解析：

- 标题
- 文本类型
- 日期
- 原始输入
- 英文改写
- 中文翻译
- 备注
- 标签
- 图片

如果有原始输入，保存后会进入原始意图高亮界面。

## 6. 原始意图

原始意图指的是：

```text
我本来想表达，但不知道英文怎么自然说的地方
```

在原始输入里选中一小段内容，按 `Space` 或点击保存，即可保存为原始意图。

如果选中的英文没有覆盖完整单词，OIO 会自动扩展到完整单词。

原始意图可以备注。如果不备注，默认会用选中的内容作为备注。

## 7. 高亮英文 Chunk

进入文本详情页后，在英文改写里选中值得学习的表达块。

可以：

```text
选中表达 -> 按 Space
```

也可以点击：

```text
挖空选中内容
```

OIO 会先保存这个 chunk。如果还有未匹配的原始意图，会先弹出匹配窗口；选好后再进入挖空界面。如果原始意图都已经匹配完，会直接进入挖空界面。

如果在挖空界面点击 `取消`，刚才新高亮的 chunk 会一起取消，不会留下无用高亮。

## 8. 挖空生成卡片

在挖空界面里：

- 单击单词：选择 / 取消，用于多词挖空
- 双击单词：直接生成卡片并保存
- `Ctrl+Z`：撤销上一步选择；如果刚刚生成了卡片，则撤销这次卡片制作
- 保存按钮：适合多词表达或需要手动调整挖空内容

如果刚刚从文本中高亮 chunk 并生成了卡片，随后按 `Ctrl+Z`，OIO 会同时撤销卡片和这个新 chunk，回到高亮前。

如果是从已有 chunk 里再次挖空，取消或撤销不会删除原来的 chunk，避免误删已经整理过的内容。

适合挖空的表达例子：

```text
in the mood for
on the fence about
get carried away
```

生成卡片时可以勾选：

```text
重要
```

重要卡片会在复习排序中优先出现。

## 9. 复习

进入 `复习` 页面，可以选择：

- 复习全部
- 只复习重要
- 每组数量：5 / 7 / 10 / 15 / 20

复习流程：

```text
看挖空句
-> 输入答案
-> 检查拼写
-> Again / Good / Easy
```

评分含义：

- Again：没有掌握，稍后再次出现
- Good：掌握一次，之后再复习
- Easy：很熟，会拉长间隔

Good / Easy 累计 3 次后，卡片会毕业，不再进入普通复习。Again 不计入成功次数。

复习排序会优先考虑：

```text
重要卡片
-> 掌握次数少的卡片
-> 复习次数少的卡片
-> 新学内容
-> 更早到期的卡片
```

## 10. 复习上下文

复习时可以看到：

- 当前挖空句
- 英文上下文
- 文本图片
- 原始意图
- 可选显示原始输入

上下文会隐藏答案，避免提前泄露。

这点很重要：OIO 的复习不是让你孤立背答案，而是在“当时那个语境”里重新调用表达。

## 11. 签到和复习记录

首页会显示：

- 今日复习
- 连续复习天数
- 签到日历

每天至少复习 1 张卡片，就会自动签到。

点击日历中的某一天，会先看到当天复习了哪些文本组。点击某个文本组后，可以查看那一组当天复习过哪些 chunks。

这样以后可以继续扩展成按组复盘、按组导出或按组生成学习材料。

## 12. 图片

每篇文本可以上传一张或多张图片。

图片适合保存：

- 截图
- 聊天记录
- 网页材料
- 书页
- 生活场景照片

图片会显示在：

- 文本详情页
- 新建 / 编辑文本窗口
- 复习上下文
- 导出内容

注意：当前图片保存在 `localStorage`，图片太多会让浏览器本地数据变大。

## 13. 搜索

文本库支持搜索。

搜索范围可以选择：

- 全部
- 标签
- 文本
- 标题

其中：

```text
全部 = 标题 + 原文 + 改写 + 翻译 + 备注 + 标签
文本 = 原文 + 改写 + 翻译 + 备注
```

## 14. 导出

文本库可以批量导出。

操作：

```text
勾选文本
-> 选择格式
-> 选择字段
-> 复制导出内容 / 下载文件
```

支持格式：

- Markdown
- HTML
- 纯文本

支持字段：

- 标题
- 日期
- 原文
- 改写
- 翻译
- 图片

下载文件规则：

- Markdown -> `.md`
- HTML -> `.html`
- 纯文本 -> `.txt`

如果需要导出图片，优先推荐 HTML。

## 15. 一个完整使用案例

假设你今天想表达：

```text
我最近有点想重新开始运动，但每次下班都觉得太累了，就一直拖着。
```

你可以这样用 OIO：

1. 在 `Prompts` 页面把这句话作为原始输入，选择 `dialogue`、英式或美式英语、合适的 CEFR 难度，然后复制提示词发给 ChatGPT。
2. 把 ChatGPT 返回的 OIO 格式内容粘贴到新建文本的 `AI 输出`，点击自动解析。如果你常用这个流程，可以勾选 `解析后自动保存`。
3. 保存后，在原始输入里高亮“重新开始运动”“下班太累”“一直拖着”。这些就是你真实想说但不确定怎么说的原始意图。
4. 进入英文改写后，找到对应表达，比如 `get back into working out`、`feel wiped out after work`、`keep putting it off`。
5. 选中一个完整 chunk，按 `Space`，先匹配对应原始意图，再进入挖空界面。
6. 如果只想练 `putting it off`，可以双击相关单词快速生成卡片；如果想练完整表达，可以单击多个词后保存。
7. 复习时，你看到的不是孤立词组，而是你当时那段关于运动和拖延的上下文。这样下次真的想说这件事时，更容易把表达调出来。

这个案例体现的是 OIO 的核心：从自己的真实表达出发，把 AI 给出的自然英文拆成可复习的 chunk，再回到自己的真实生活里使用。

## 16. 当前限制

当前版本仍然是本地静态版：

```text
index.html
styles.css
app.js
localStorage
```

限制：

- 没有云同步
- 没有自动备份
- 图片多了可能占用较多 localStorage
- 换浏览器可能看不到原来的数据

后续可以优先考虑：

- 数据备份 / 导入
- IndexedDB
- 按组导出
- 图片导出增强
- 音频 / 听音练习
- API 自动改写
