# OIO 复习体验临时修改方案

## 1. 修正复习高亮显示

### 当前问题

现在复习页在卡片下面额外显示了完整语境，并把 chunk 高亮出来。

问题是：

- 如果高亮 chunk 就是答案，用户会提前看到答案
- 这会破坏挖空复习的测试效果

### 修改方向

复习时不要在下方直接显示包含答案的完整高亮句。

改成：

```text
只在挖空句本身显示高亮位置
不要额外暴露答案文本
```

例如：

```text
I'm not really [___ ___ ___ ___] small talk today.
```

高亮的是空的位置，而不是答案。

## 2. 复习时支持滚动查看上下文

### 目标

复习时可以看到这张卡片所属文本的上下文，但不能提前看到答案。

### 设计原则

- 可以查看当前卡片所在文本的前后内容
- 当前答案部分必须保持隐藏或挖空
- 长文本支持滚动
- 上下文用于帮助回忆语境，不用于泄露答案

### 交互设计

复习页增加一个上下文区域：

```text
[当前卡片]

I'm not really [___ ___ ___ ___] small talk today.

[可滚动上下文]
上一句……
当前句：I'm not really [___ ___ ___ ___] small talk today.
下一句……
后续内容……
```

上下文区域要求：

- 高度固定
- 内容可滚动
- 当前卡片所在句突出显示
- 当前答案在上下文中也显示为挖空，而不是原词

## 3. 增加打字验证拼写功能

### 当前复习方式

```text
看挖空句
-> 心里回忆
-> 点显示答案
-> 自己判断 Again / Good / Easy
```

### 新增方式

增加一个输入框，让用户直接输入答案。

流程：

```text
显示挖空句
-> 用户在输入框输入答案
-> 点击“检查”
-> 系统对比用户输入和正确答案
-> 显示结果
-> 用户再选择 Again / Good / Easy
```

### 对比规则

第一版先做简单对比：

```text
忽略首尾空格
忽略大小写
保留单词拼写和顺序
```

例如：

正确答案：

```text
in the mood for
```

用户输入：

```text
In the mood for
```

判定：

```text
正确
```

用户输入：

```text
in mood for
```

判定：

```text
错误
```

## 4. 高亮选择自动扩展到完整单词

### 当前问题

用户高亮 chunk 时，可能只选中了半个单词。

例如原句：

```text
I'm not really in the mood for small talk today.
```

用户可能只选中：

```text
moo
```

这会导致 chunk 和后续挖空都不完整。

### 修改方向

当用户确认创建 chunk 时，如果选区边界落在一个单词中间，系统自动扩展到完整单词。

例如：

```text
moo
```

自动扩展为：

```text
mood
```

如果用户选中：

```text
the moo
```

自动扩展为：

```text
the mood
```

### 边界规则

第一版可以按英文学习场景处理：

```text
单词字符：A-Z、a-z、0-9、撇号 '
边界字符：空格、标点、换行
```

扩展逻辑：

```text
1. 找到用户选中文本在完整文本中的位置
2. 如果选区左边还是单词字符，就继续向左扩展
3. 如果选区右边还是单词字符，就继续向右扩展
4. 用扩展后的完整文本作为 chunk
```

示例：

```text
don't
```

如果只选到：

```text
don
```

应扩展成：

```text
don't
```

## 5. 复习页状态设计

卡片复习状态可以分为：

```ts
type ReviewState = {
  userAnswer: string;
  checked: boolean;
  isCorrect: boolean;
};
```

页面逻辑：

```text
未检查前：
- 显示挖空句
- 显示输入框
- 显示“检查”按钮
- 不显示答案

检查后：
- 显示 Correct / Not quite
- 显示正确答案
- 显示 Again / Good / Easy
```

## 6. 上下文安全显示逻辑

需要一个函数专门生成安全上下文：

```ts
function maskAnswerInContext(fullText, answer) {
  return fullText.replaceAll(answer, blanks);
}
```

也就是说：

- 上下文可以显示完整文本
- 但所有答案字符串都替换成挖空
- 避免提前泄露答案

## 7. 修改优先级

### 第一优先级

- 删除复习下方直接显示答案 chunk 的高亮语境
- 改成在挖空处高亮
- 上下文中隐藏答案

### 第二优先级

- 增加可滚动上下文区域
- 当前句突出显示

### 第三优先级

- 增加输入框
- 增加检查按钮
- 增加拼写验证结果

### 第四优先级

- 高亮创建 chunk 时自动扩展到完整单词

## 8. 最终复习页结构

```text
标题：语境复习
进度：2 / 5

卡片：
I'm not really [___ ___ ___ ___] small talk today.

输入答案：
[________________________]

按钮：
[检查]

检查后：
Correct / Not quite
正确答案：in the mood for

评分：
[Again] [Good] [Easy]

上下文：
一个固定高度、可滚动的文本区域。
当前句中的答案保持挖空。
```

## 9. 暂不做

第一版先不做：

- 模糊拼写容错
- 语义判断
- AI 判断答案是否自然
- 多答案
- 自动发音
- 输入时实时纠错
